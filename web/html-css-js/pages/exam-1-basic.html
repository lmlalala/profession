1 HTML
HTML5 语义化标签有哪些？为什么要使用语义化？
参考答案：
header 、 nav 、 main 、 article 、 section 、 aside 、 footer 等。好处：① 提升 SEO，搜索引擎更好理解页面结构 ② 增强可访问性，屏幕阅读器可正确朗读 ③ 代码可读性和可维护性更好。
2 HTML
localStorage 、 sessionStorage 和 Cookie 的区别？
参考答案：
localStorage 约 5MB、持久存储、不随请求发送； sessionStorage 约 5MB、会话级别（关闭标签页清除）、不随请求发送； Cookie 约 4KB、可设过期时间、每次请求自动携带、可设 HttpOnly / Secure / SameSite 。
3 HTML
Canvas 和 SVG 的区别？分别适用于什么场景？
参考答案：
Canvas 基于像素（位图），通过 JS API 绘制，适合游戏、图表、图片处理等高性能场景；SVG 基于矢量（XML），支持 DOM 操作和 CSS 样式，适合图标、地图、数据可视化等需要缩放和交互的场景。Canvas 不支持事件绑定到图形元素，SVG 每个元素都是 DOM 节点可绑定事件。
4 CSS
请解释 CSS 盒模型， content-box 和 border-box 的区别？
参考答案：
盒模型由 content + padding + border + margin 组成。 content-box （默认）： width 只包含内容区，实际占宽 = width + padding + border； border-box ： width 包含 content + padding + border，实际占宽 = width。推荐全局设置 * { box-sizing: border-box; } 。
5 CSS
什么是 BFC？如何触发？有什么作用？
参考答案：
BFC（Block Formatting Context）是独立的渲染区域，内部布局不影响外部。触发方式： overflow: hidden/auto/scroll 、 display: flex/grid/inline-block 、 position: absolute/fixed 、 float: left/right 。作用：① 清除浮动（父元素高度塌陷）② 阻止 margin 合并 ③ 阻止元素被浮动元素覆盖。
6 CSS
实现水平垂直居中有哪些方案？
参考答案：
① Flex： display:flex; justify-content:center; align-items:center
② Grid： display:grid; place-items:center
③ absolute + transform： position:absolute; top:50%; left:50%; transform:translate(-50%,-50%)
④ absolute + margin:auto： position:absolute; inset:0; margin:auto （需固定宽高）。
7 CSS
CSS 选择器优先级如何计算？
参考答案：
!important > 内联样式 (1000) > ID (100) > 类/伪类/属性 (10) > 标签/伪元素 (1) > 通配符 (0)。同优先级后写的覆盖先写的。
8 CSS
Flexbox 和 Grid 的区别？分别适用于什么场景？
参考答案：
Flexbox 是一维布局（主轴方向），适合导航栏、工具栏、卡片列表等线性排列；Grid 是二维布局（行+列），适合页面整体布局、仪表盘、复杂网格。Flexbox 更灵活适合内容驱动，Grid 更精确适合布局驱动。
9 CSS
什么是重排（Reflow）、重绘（Repaint）和合成（Composite）？如何优化？
参考答案：
重排：改变几何属性（width/height/margin/padding）触发，开销最大；重绘：改变外观属性（color/background）触发，中等开销；合成：只改变 transform / opacity ，GPU 加速，开销最小。优化：① 用 transform/opacity 做动画 ② 批量修改 DOM ③ 避免频繁读取布局属性 ④ 使用 will-change 提示浏览器。
10 JS
JavaScript 有哪些数据类型？如何准确判断类型？
参考答案：
7 种原始类型： string 、 number 、 boolean 、 null 、 undefined 、 symbol 、 bigint + 1 种引用类型： object 。判断方法： typeof （注意 typeof null === 'object' ）、 instanceof （判断原型链）、 Object.prototype.toString.call() （最准确，返回 '[object Type]' ）、 Array.isArray() （判断数组）。
11 JS
解释闭包及其应用场景。
参考答案：
闭包是函数能够访问其词法作用域中变量的能力，即使函数在其词法作用域之外执行。应用：① 数据私有化（模拟私有变量）② 函数工厂（createCounter）③ 回调和事件处理 ④ 柯里化和偏函数 ⑤ 模块模式。注意：闭包会保持对外部变量的引用，可能导致内存泄漏。
12 JS
解释 this 的绑定规则，箭头函数的 this 有什么不同？
参考答案：
this 绑定优先级： new > 显式绑定（call/apply/bind）> 隐式绑定（obj.fn()）> 默认绑定（window/undefined）。箭头函数没有自己的 this ，继承外层词法作用域的 this ，且无法通过 call/apply/bind 改变。
13 JS
原型链是什么？如何实现继承？
参考答案：
每个对象都有 __proto__ 指向其构造函数的 prototype ，形成原型链。属性查找沿链向上直到 null 。继承方式：① 原型链继承 ② 借用构造函数 ③ 组合继承 ④ 寄生组合继承 ⑤ ES6 class extends （语法糖，本质是寄生组合继承）。
14 JS
var 、 let 、 const 的区别？什么是暂时性死区（TDZ）？
参考答案：
var ：函数作用域，存在变量提升，可重复声明，挂载到 window； let ：块级作用域，不可重复声明，存在 TDZ； const ：块级作用域，声明时必须赋值，不可重新赋值（引用类型内容可变）。TDZ： let / const 声明的变量在声明语句之前的区域，访问会抛出 ReferenceError 。
15 ES6
解构赋值有哪些用法？展开运算符的应用场景？
参考答案：
数组解构 [a, b, ...rest] 、对象解构 { name, age: myAge = 18 } 、参数解构、嵌套解构、交换变量 [a, b] = [b, a] 。展开运算符：合并数组 [...a, ...b] 、合并对象 { ...obj1, ...obj2 } 、函数参数收集 fn(...args) 、浅拷贝。
16 ES6
Promise 的三种状态是什么？ Promise.all 、 Promise.race 、 Promise.allSettled 的区别？
参考答案：
pending （等待）、 fulfilled （成功）、 rejected （失败），状态不可逆。 all ：全部成功才 resolve，一个失败即 reject； race ：最先 settle 的结果（不管成败）； allSettled ：全部完成后返回所有结果（包含 status 和 value/reason）； any （ES2021）：最先成功的结果。
17 ES6
async/await 的原理是什么？与 Promise 的关系？
参考答案：
async 函数返回 Promise， await 暂停执行等待 Promise resolve。本质是 Generator + 自动执行器的语法糖。 await 后的代码相当于 .then() 回调（微任务）。错误处理用 try/catch 。并发优化用 Promise.all([p1, p2]) 并行等待，而非串行 await。
18 JS
事件循环（Event Loop）的执行顺序是什么？
参考答案：
① 执行同步代码（调用栈）② 清空微任务队列（Promise.then、queueMicrotask、MutationObserver、await 后的代码）③ 取一个宏任务执行（setTimeout、setInterval、I/O、requestAnimationFrame）④ 再清空微任务队列 ⑤ 循环。
19 JS
手写一个防抖函数（debounce）。
参考答案： 
function debounce(fn, delay) {
  let timer = null;
  return function (...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), delay);
  };
}
原理：每次触发重置定时器，只有停止触发 delay 毫秒后才执行。应用：搜索框输入、窗口 resize。
20 JS
手写一个深拷贝函数，需要处理循环引用。
参考答案：
使用 WeakMap 记录已拷贝对象解决循环引用。递归处理，特殊类型（Date、RegExp、Map、Set）单独处理，用 Reflect.ownKeys 遍历包括 Symbol key。简单场景可用 structuredClone() 。
21 JS
Proxy 和 Object.defineProperty 的区别？
参考答案：
Proxy 可拦截 13 种操作（get/set/deleteProperty/has/ownKeys 等），支持数组下标和新增属性；defineProperty 只能劫持已有属性的 get/set。Proxy 是懒代理（访问时才递归），defineProperty 需要递归遍历所有属性。Proxy 是 Vue3 响应式核心，defineProperty 是 Vue2 的。
22 JS
什么是事件委托？有什么优势？
参考答案：
利用事件冒泡，将子元素的事件监听器绑定到父元素上，通过 event.target 判断实际触发元素。优势：① 减少事件监听器数量，节省内存 ② 动态添加的子元素自动拥有事件处理 ③ 简化代码。
23 JS
WeakMap 和 Map 的区别？ WeakRef 的作用？
参考答案：
Map 的 key 可以是任意类型，强引用不会被 GC；WeakMap 的 key 只能是对象，弱引用不阻止 GC，不可遍历。WeakMap 适合缓存、私有数据存储。WeakRef（ES2021）创建对对象的弱引用， deref() 获取对象或 undefined （已被 GC）。
24 JS
常见的设计模式有哪些？分别适用于什么场景？
参考答案：
① 单例模式：全局唯一实例（弹窗、Store）② 观察者模式：一对多依赖（EventEmitter）③ 发布订阅：有中间调度层（EventBus）④ 策略模式：算法族封装（表单验证规则）⑤ 工厂模式：创建对象（组件工厂）⑥ 装饰器模式：增强功能（HOC、AOP）。
25 JS
前端有哪些常见的内存泄漏场景？如何排查？
参考答案：
① 意外全局变量 ② 未清除的定时器/事件监听 ③ 闭包持有大对象引用 ④ 脱离 DOM 的引用 ⑤ console.log 保留引用。排查：Chrome DevTools → Memory → Heap Snapshot 对比，或 Performance Monitor 观察 JS Heap Size 趋势。
