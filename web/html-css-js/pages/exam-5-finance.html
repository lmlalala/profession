试卷五：金融/支付中台方向

Vue3 + TS 深度 · 金融安全 · 大数精度 · 高并发 · 权限控制 · 综合实战 · 共 30 题 · 建议先作答再对照参考答案

1 Vue3
Vue3 的响应式原理？对比 Vue2 的 Object.defineProperty 有哪些核心优势？
参考答案：
Vue3 使用 Proxy 拦截 13 种操作。核心优势：① 支持新增/删除属性检测（Vue2 需 Vue.set）② 支持数组下标和 length 变化 ③ 懒代理（访问时才递归代理，Vue2 初始化递归所有）④ 支持 Map/Set/WeakMap ⑤ 更好的性能。依赖收集：WeakMap<target, Map<key, Set<effect>>> 三层结构。
2 Vue3
ref 和 reactive 的区别？解构 reactive 时如何保证响应式不丢失？
参考答案：
ref 包装任意类型（.value 访问），reactive 只能包装对象。解构 reactive 丢失响应式，解决：toRefs(state) 将每个属性转为 ref，或 toRef(state, 'key')。ref 在模板中自动解包，reactive 不需要 .value。
3 Vue3
setup 语法的优势？watch 和 watchEffect 的核心区别？
参考答案：
script setup 优势：更简洁（无需 return）、更好的 TS 推断、编译优化。watch：显式指定监听源，可获取新旧值，默认懒执行；watchEffect：自动追踪依赖，立即执行，无旧值。watch 适合对比变化，watchEffect 适合副作用同步。
4 Vue3
Vue3 生命周期对比 Vue2？onMounted 和 nextTick 的区别？
参考答案：
beforeCreate/created → setup；mounted → onMounted；beforeDestroy → onBeforeUnmount。onMounted：组件挂载后执行一次；nextTick：DOM 更新后执行（每次响应式数据变化触发 DOM 更新后）。nextTick 返回 Promise，可 await。
5 Vue3
defineAsyncComponent 和 Suspense 如何实现按需加载？
参考答案：
defineAsyncComponent(() => import('./Comp.vue')) 定义异步组件。Suspense 包裹异步组件，#default 显示加载完成内容，#fallback 显示加载中状态。支持嵌套异步组件和 async setup()。
6 Vue3
Teleport 和 Fragment 的理解和应用？
参考答案：
Teleport：将组件内容传送到 DOM 其他位置（如 body），适合模态框、通知、下拉菜单等需要脱离父级 overflow/z-index 限制的场景。Fragment：组件支持多根节点，减少无意义的包裹 div。
7 TS
TypeScript 和 JavaScript 的核心区别？为什么大型项目优先选择 TS？
参考答案：
TS 是 JS 的超集，增加静态类型系统。优势：① 编译时发现类型错误 ② IDE 智能提示和自动补全 ③ 代码即文档（类型注解）④ 重构安全（类型检查保障）⑤ 团队协作效率提升。大型项目接口多、人员多，类型系统能显著降低沟通成本和 bug 率。
8 TS
interface 和 type 的区别？适用场景？
参考答案：
interface：可声明合并、extends 继承、class implements。type：支持联合/交叉/条件/映射类型。推荐：对象形状用 interface（可扩展），联合类型/工具类型用 type。性能上 interface 略优（编译器内部优化）。
9 TS
泛型在支付中台的应用？举例统一请求响应封装。
参考答案：
interface ApiResponse<T> { code: number; data: T; message: string } + async function request<T>(url: string, params?: any): Promise<ApiResponse<T>>。使用：const { data } = await request<TransactionList>('/api/transactions')。泛型保证响应数据类型安全，IDE 自动补全字段。
10 TS
any、unknown、never 的区别？如何避免滥用 any？
参考答案：
any 跳过检查（传染性强），unknown 安全的 any（使用前必须收窄），never 永不存在的值。避免 any：① 开启 noImplicitAny ② 用 unknown + 类型守卫替代 ③ 第三方库缺类型时写 .d.ts ④ 复杂类型用泛型抽象 ⑤ Code Review 检查。
11 TS
类型收窄的方式？结合 in、typeof、自定义类型守卫举例。
参考答案：
typeof：if (typeof x === 'string')；in：if ('swim' in animal)；自定义守卫：function isTransaction(x: unknown): x is Transaction { return typeof x === 'object' && x !== null && 'amount' in x }。可辨识联合：type Result = { status: 'success'; data: T } | { status: 'error'; message: string }。
12 TS
金融场景如何定义复杂数据类型？
参考答案：
交易流水：interface Transaction { readonly id: string; amount: string; currency: 'CNY'|'USD'; status: TransactionStatus; createdAt: string; metadata?: Record<string, unknown> }。金额用 string 避免精度丢失，id 设 readonly，可选属性用 ?。对账结果用可辨识联合区分成功/失败。
13 工程
Vite 开发环境速度快的核心原理？
参考答案：
① 浏览器原生 ESM：不打包，按需编译（请求时才转换模块）② esbuild 预构建：Go 编写，将 CJS 依赖转为 ESM，速度比 JS 快 10-100 倍 ③ HMR 精准更新：只更新变更模块，不重建整个 bundle ④ 依赖缓存：node_modules/.vite 缓存预构建结果。
14 工程
如何配置多环境（开发/测试/生产）？
参考答案：
Vite：.env.development / .env.staging / .env.production，变量以 VITE_ 前缀。package.json scripts："build:staging": "vite build --mode staging"。代码中 import.meta.env.VITE_API_URL。Webpack：DefinePlugin + env 文件。
15 工程
HMR 机制是什么？如何自定义处理？
参考答案：
Hot Module Replacement：模块变更时，只替换变更模块而不刷新页面。Vite HMR 基于 ESM，通过 WebSocket 通知客户端。自定义：if (import.meta.hot) { import.meta.hot.accept('./module.js', (newModule) => { /* 处理更新 */ }) }。

17 工程
Webpack loader 和 plugin 的区别？各举 3 个常用的。
参考答案：
loader 转换文件：babel-loader（ES6→ES5）、css-loader（处理 CSS 导入）、file-loader（处理文件资源）。plugin 扩展功能：HtmlWebpackPlugin（生成 HTML）、MiniCssExtractPlugin（提取 CSS 文件）、DefinePlugin（定义环境变量）。loader 在 module.rules 配置，plugin 在 plugins 数组。
18 业务
如何将复杂金融业务（清算、对账）抽象为可维护的前端代码？
参考答案：
① 领域模型抽象：定义清晰的 TypeScript 类型（交易、账户、对账记录）② 状态机管理：用有限状态机描述业务流程（待处理→处理中→成功/失败）③ 策略模式：不同清算规则封装为策略 ④ 组件化：业务组件（交易列表、对账面板）+ 通用组件分离 ⑤ 配置化：规则引擎驱动，减少硬编码。
19 业务
高并发交易系统，前端如何做防抖、节流和重试？如何保证安全性？
参考答案：
防抖：提交按钮 loading 状态 + 接口去重（取消重复请求）。节流：限制操作频率。重试：指数退避重试（1s→2s→4s），设置最大重试次数。安全性：① 请求签名（timestamp+nonce+sign）② 幂等性（唯一请求 ID）③ HTTPS ④ Token 认证 ⑤ 关键操作二次验证。
20 业务
金融表单校验（金额、卡号、身份证）如何实现？
参考答案：
金额：正则 + BigNumber 精度校验，限制小数位数。卡号：Luhn 算法校验 + 银行卡 BIN 识别。身份证：18 位格式校验 + 校验码算法 + 生日合法性。实现：自定义校验规则 + Element Plus 的 FormItem rules，支持异步校验（如卡号实时验证）。

22 安全
金融场景 XSS 防御？如何处理富文本安全？
参考答案：
① 输出编码：HTML 实体转义（&lt; &gt; &amp;）② CSP 策略：限制脚本来源 ③ HttpOnly Cookie：防止 JS 读取 ④ 富文本：白名单过滤（DOMPurify），只允许安全标签和属性 ⑤ 避免 innerHTML，用 textContent ⑥ 输入长度限制。
23 安全
CSRF 攻击防御？Axios 中如何统一配置 Token？
参考答案：
防御：① CSRF Token（服务端种到 Cookie，前端读取放到请求头 X-CSRF-Token）② SameSite=Strict Cookie ③ 验证 Referer/Origin。Axios 配置：axios.interceptors.request.use(config => { config.headers['X-CSRF-Token'] = getCookie('csrf_token'); return config })。
24 安全
敏感数据（银行卡号、密码）如何加密传输？
参考答案：
① HTTPS 传输层加密 ② 密码：前端 RSA 公钥加密，后端私钥解密 ③ 银行卡号：脱敏显示（6222****1234），传输时加密 ④ 前端不存储明文敏感数据 ⑤ Web Crypto API 进行加密操作 ⑥ 密钥管理：公钥从服务端获取，不硬编码。
25 安全
如何实现 RBAC 权限控制？
参考答案：
模型：用户→角色→权限。实现：① 登录后获取角色和权限列表 ② 路由权限：根据权限动态 addRoute ③ 菜单权限：过滤菜单树 ④ 按钮权限：自定义指令 v-permission="'order:delete'" ⑤ 接口权限：请求拦截器携带 Token，401 跳转登录。
26 性能
Web Vitals 核心指标？针对金融中台如何优化？
参考答案：
LCP < 2.5s：SSR 首屏、关键资源预加载、图片优化。INP < 200ms：代码分割减小 JS 体积、Web Worker 处理复杂计算。CLS < 0.1：图表/表格预留尺寸、字体 font-display: swap。金融中台重点：大数据表格虚拟滚动、图表按需加载、接口请求合并。

29 业务
金融场景大数字精度问题如何解决？
参考答案：
JS Number 是 IEEE 754 双精度浮点，最大安全整数 2^53-1。解决：① 金额用字符串传输和存储 ② 计算用 BigNumber.js / decimal.js ③ 后端返回 string 类型金额 ④ 展示时格式化（千分位、小数位）⑤ 比较用 BigNumber 的比较方法而非 === 。
