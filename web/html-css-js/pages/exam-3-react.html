1 React
React 的虚拟 DOM 是什么？为什么需要它？
参考答案：
虚拟 DOM 是用 JS 对象描述真实 DOM 的轻量表示。好处：① 批量更新，减少直接 DOM 操作 ② 跨平台（React Native）③ diff 算法找出最小变更集 ④ 声明式编程，开发者只关心状态。
2 React
JSX 的本质是什么？编译后变成什么？
参考答案：
JSX 是 React.createElement() 的语法糖。<div className="box">Hello</div> 编译为 React.createElement('div', {className:'box'}, 'Hello')，返回 VNode 对象。React 17+ 使用新的 JSX Transform，不再需要显式 import React。
3 React
React Fiber 架构解决了什么问题？
参考答案：
解决 React 15 Stack Reconciler 的同步递归渲染导致的长时间阻塞主线程问题。Fiber 将渲染工作拆分为小单元（fiber 节点），通过 requestIdleCallback 实现可中断渲染和时间切片。每个 fiber 节点有 child/sibling/return 指针形成链表，支持暂停、恢复和优先级调度。
4 React
React diff 算法的策略是什么？
参考答案：
① 同层比较，不跨层级 ② 不同类型的元素直接替换整棵子树 ③ 同类型元素通过 key 标识复用。多子节点 diff：两轮遍历，第一轮处理位置不变的节点，第二轮用 Map 处理移动/新增/删除，通过 lastPlacedIndex 判断是否需要移动。
5 React
useState 和 useReducer 的区别？
参考答案：
useState 适合简单状态（单值），useReducer 适合复杂状态逻辑（多个相关状态、复杂更新逻辑）。useReducer 类似 Redux 的 dispatch/action/reducer 模式，更易测试。useState 内部其实是 useReducer 的特殊情况。
6 React
useEffect 的执行时机和清理机制？
参考答案：
useEffect 在 DOM 更新后异步执行（不阻塞渲染）。依赖数组：[] 只执行一次，[dep] 依赖变化时执行，不传则每次渲染后执行。返回清理函数在组件卸载或下次 effect 执行前调用。useLayoutEffect 在 DOM 更新后同步执行（阻塞渲染），适合需要读取 DOM 布局的场景。
7 React
useMemo 和 useCallback 的区别？什么时候使用？
参考答案：
useMemo 缓存计算结果值，useCallback 缓存函数引用。使用场景：① 传给子组件的 props（配合 React.memo 避免不必要渲染）② 昂贵的计算（useMemo）③ 作为其他 Hook 的依赖。不要过度使用，简单场景的缓存开销可能大于重新计算。
8 React
useRef 的作用？与 useState 的区别？
参考答案：
useRef 创建可变引用，.current 值变化不触发重渲染。用途：① 访问 DOM 元素 ② 保存跨渲染的可变值（定时器 ID、上一次的值）③ 避免闭包陷阱。与 useState 区别：ref 变化不触发渲染，同步更新；state 变化触发渲染，异步批量更新。
9 React
React.memo、useMemo、useCallback 如何配合优化性能？
参考答案：
React.memo 包裹子组件，只在 props 变化时重渲染。父组件用 useCallback 缓存传给子组件的函数 props，用 useMemo 缓存传给子组件的对象/数组 props。三者配合才能有效避免不必要的子组件渲染。
10 React
Context 的使用场景和性能问题？
参考答案：
Context 用于跨组件层级传递数据（主题、语言、用户信息）。性能问题：Context value 变化会导致所有消费者重渲染，即使消费的值没变。解决：① 拆分 Context ② useMemo 包裹 value ③ 使用状态管理库（Redux/Zustand）代替。
11 React
Redux Toolkit 的核心概念？

13 React
Hooks 的规则和原理？为什么不能在条件语句中使用？
参考答案：
规则：① 只在函数组件顶层调用 ② 不在循环/条件/嵌套函数中调用。原理：Hooks 存储在 fiber.memoizedState 链表中，按调用顺序索引。如果条件调用导致顺序变化，Hook 会对应到错误的状态。
14 React
如何自定义 Hook？举例说明。
参考答案：
自定义 Hook 是以 use 开头的函数，内部可调用其他 Hook。例： 
function useLocalStorage(key, initial) {
  const [value, setValue] = useState(() =>
    JSON.parse(localStorage.getItem(key)) ?? initial);
  useEffect(() => {
    localStorage.setItem(key, JSON.stringify(value));
  }, [key, value]);
  return [value, setValue];
}

16 React
Server Components 是什么？与 SSR 的区别？
参考答案：
Server Components 在服务端渲染，不发送 JS 到客户端（零 bundle size），可直接访问数据库/文件系统。与 SSR 区别：SSR 仍然发送完整 JS bundle 用于 hydration；Server Components 不需要 hydration，客户端只接收渲染结果。可与 Client Components 混合使用。

18 React
如何处理 React 中的错误？
参考答案：
① Error Boundary（类组件 componentDidCatch/getDerivedStateFromError）捕获渲染错误 ② try/catch 捕获事件处理和异步错误 ③ React Router errorElement 处理路由错误 ④ Suspense fallback 处理加载错误。注意：Error Boundary 不能捕获事件处理器、异步代码、SSR 中的错误。
19 Eng
Next.js 的渲染模式有哪些？
参考答案：
① SSR（Server-Side Rendering）：每次请求服务端渲染 ② SSG（Static Site Generation）：构建时生成静态 HTML ③ ISR（Incremental Static Regeneration）：静态页面定时重新生成 ④ CSR（Client-Side Rendering）：纯客户端渲染。App Router 默认 Server Components，可按组件粒度选择渲染模式。
20 Eng
Next.js App Router 与 Pages Router 的区别？
参考答案：
App Router：① 基于 React Server Components ② 文件系统路由（app/ 目录）③ layout.tsx 嵌套布局 ④ loading.tsx/error.tsx 自动处理 ⑤ Server Actions 替代 API Routes。Pages Router：传统 CSR/SSR 模式，getServerSideProps/getStaticProps 数据获取。

23 React
什么是受控组件和非受控组件？
参考答案：
受控组件：表单值由 React state 控制（value + onChange），数据流单向。非受控组件：表单值由 DOM 自身管理，通过 ref 获取值（defaultValue）。推荐使用受控组件，数据可预测可验证。
