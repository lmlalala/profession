1. BFC 块级格式化上下文 面试必考

BFC 是什么？ Block Formatting Context，独立的渲染区域，内部布局不影响外部。
触发 BFC 的方式： overflow: hidden/auto/scroll | display: flex/grid/inline-block | position: absolute/fixed | float: left/right

BFC 的三大作用 — 编辑代码查看效果

① 清除浮动（父元素高度塌陷）

未触发 BFC → 高度塌陷

② 触发 BFC 后

overflow:hidden 触发 BFC → 高度自适应

③ 阻止 margin 合并

margin-bottom: 20px

margin-top: 30px（BFC 包裹后不合并）

h4 { font-size: 0.85rem; color: #8892b0; margin: 1rem 0 0.4rem; } /* 浮动元素 */ .float-box { float: left; width: 60px; height: 60px; background: #6c63ff; border-radius: 4px; margin-right: 10px; } /* 父元素 */ .parent { background: rgba(255,255,255,0.05); border: 2px dashed #2e3350; padding: 8px; border-radius: 4px; margin-bottom: 0.5rem; } /* 未触发 BFC：高度塌陷 */ .no-bfc { border-color: #ff6584; } /* 触发 BFC：overflow:hidden */ .with-bfc { overflow: hidden; /* 触发 BFC */ border-color: #00d4aa; } .tip { font-size: 0.8rem; color: #8892b0; } /* margin 合并演示 */ .margin-demo { margin-top: 0.5rem; } .m1 { margin-bottom: 20px; background: rgba(108,99,255,0.1); padding: 6px; border-radius: 4px; font-size: 0.8rem; } .m2 { margin-top: 30px; background: rgba(0,212,170,0.1); padding: 6px; border-radius: 4px; font-size: 0.8rem; } /* BFC 包裹阻止 margin 合并 */ .bfc-wrapper { overflow: hidden; }

触发层叠上下文的条件： position + z-index（非auto） | opacity < 1 | transform | filter | will-change | display: flex/grid 的子元素 + z-index
层叠顺序（从低到高）
背景/边框 → 负z-index → 块级盒子 → 浮动盒子 → 行内盒子 → z-index:0 → 正z-index

常见陷阱 子元素的 z-index 只在同一层叠上下文内比较。父元素 z-index 低，子元素无论设多高都无法超过其他父元素的子元素。

4. 响应式设计 面试高频

/* 媒体查询 */ @media (max-width: 768px) { .container { flex-direction: column; } } @media (min-width: 768px) and (max-width: 1024px) { .grid { grid-template-columns: repeat(2, 1fr); } } /* 视口单位 */ width: 100vw; /* 视口宽度 100% */ height: 100vh; /* 视口高度 100% */ font-size: 5vw; /* 随视口宽度缩放 */ /* clamp：响应式字体大小 */ font-size: clamp(1rem, 2.5vw, 2rem); /* min, preferred, max */ /* 图片响应式 */ img { max-width: 100%; height: auto; }
移动优先（Mobile First） 先写手机样式，再用 min-width 媒体查询覆盖大屏样式。比"桌面优先"性能更好，因为移动端不需要下载和解析覆盖样式。

7. CSS 性能优化 面试高频

优化点说明
使用 transform/opacity 做动画只触发 Composite，不触发 Reflow/Repaint
will-change: transform提前告知浏览器该元素会变化，创建独立图层（慎用，耗内存）
避免使用 @import串行加载，改用 <link> 并行加载
减少选择器嵌套层级浏览器从右向左匹配，层级越深越慢
使用 contain: layout限制重排范围，提升性能
关键 CSS 内联首屏关键样式内联，减少渲染阻塞
避免频繁读取布局属性offsetWidth/scrollTop 等会强制同步布局（强制重排）
重排（Reflow）vs 重绘（Repaint）vs 合成（Composite） 重排 > 重绘 > 合成（性能开销从高到低）。 修改 width/height/margin/padding/top/left → 重排。 修改 color/background/visibility → 重绘。 修改 transform/opacity → 合成（最快）。
