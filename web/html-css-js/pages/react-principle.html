2. Fiber 架构 面试必考

为什么需要 Fiber？ React 15 的 Stack Reconciler 是同步递归的，一旦开始无法中断，大型组件树会阻塞主线程导致卡顿。 React 16 引入 Fiber，将渲染工作拆分为可中断的小单元，利用浏览器空闲时间（requestIdleCallback）执行。

3. React diff 算法 面试必考

React diff 三大策略： ① 同层比较（不跨层）； ② 类型不同直接替换； ③ key 相同则复用（key 的重要性！）
/** * React 多节点 diff（lastPlacedIndex 算法，简化版）。 * 策略：两轮遍历 * 第一轮：顺序扫描，key 相同则复用，key 不同立即退出 * 第二轮：剩余旧节点建立 key→index 映射表，遍历剩余新节点 * 用 lastPlacedIndex 判断是否需要移动 * * @param {string[]} oldList - 旧子节点 key 列表 * @param {string[]} newList - 新子节点 key 列表 * @returns {{ ops: string[], moves: number }} 操作日志与移动次数 */ function reactDiff(oldList, newList) { const ops = [] let lastPlacedIndex = 0 // 最后一个「不需要移动」的旧节点下标 let newIdx = 0 let oldIdx = 0 // ── 第一轮：顺序遍历，处理前缀 key 相同的节点 ── while (oldIdx < oldList.length && newIdx < newList.length) { if (oldList[oldIdx] !== newList[newIdx]) break // key 不同，退出第一轮 ops.push(`复用 ${newList[newIdx]}（第一轮，index=${oldIdx}）`) lastPlacedIndex = oldIdx oldIdx++; newIdx++ } // ── 第二轮：剩余旧节点建立 key→oldIndex 映射表 ── const existingMap = new Map() for (let i = oldIdx; i < oldList.length; i++) { existingMap.set(oldList[i], i) } let moves = 0 // 遍历剩余新节点 for (; newIdx < newList.length; newIdx++) { const key = newList[newIdx] const oldIndex = existingMap.get(key) if (oldIndex === undefined) { // 旧列表中不存在：新增 ops.push(`新增 ${key}`) } else { existingMap.delete(key) if (oldIndex < lastPlacedIndex) { // 旧位置在 lastPlacedIndex 之前：需要向后移动 ops.push(`移动 ${key}（oldIndex=${oldIndex} < lastPlacedIndex=${lastPlacedIndex}）`) moves++ } else { // 旧位置在 lastPlacedIndex 之后或相等：不需要移动，原地复用 ops.push(`复用 ${key}（第二轮，oldIndex=${oldIndex}，无需移动）`) lastPlacedIndex = oldIndex } } } // 剩余旧节点全部删除 existingMap.forEach((_, key) => ops.push(`删除 ${key}`)) return { ops, moves } } // 测试 1：React 经典场景（lastPlacedIndex 的体现） const r1 = reactDiff(['A','B','C','D'], ['D','A','B','C']) console.log('=== 测试1：D 移到最前 ===') console.log('旧: A B C D → 新: D A B C') r1.ops.forEach(op => console.log(' ', op)) console.log(`移动次数: ${r1.moves}（React 会移动 A B C，而非只移动 D！）`) // 测试 2：有新增和删除 const r2 = reactDiff(['A','B','C','D','E'], ['D','A','C','B','F']) console.log('\n=== 测试2：综合场景 ===') console.log('旧: A B C D E → 新: D A C B F') r2.ops.forEach(op => console.log(' ', op)) console.log(`移动次数: ${r2.moves}`) // 对比：Vue2 双端 vs React lastPlacedIndex console.log('\n=== 关键区别 ===') console.log('React 不用双端指针，只从左到右扫描一遍') console.log('lastPlacedIndex 保证：旧位置在其右侧的节点不需要移动') console.log('缺点：[A,B,C,D]→[D,A,B,C] React 移动 A/B/C，Vue2 只移动 D')

4. useState 原理 面试必考

useState 本质：每个组件对应一个 Fiber，Fiber 上有一个 hooks 数组， 每次调用 useState 按顺序占用一个槽位。setState 将新值入队，然后触发组件重新渲染， 下次渲染时从同一槽位读取最新值。
// ── 第一步：useState 的数据结构 ───────────────────────────── // 每个 Fiber（组件实例）维护一个 hooks 数组 // 每次调用 useState 按顺序对应数组中的一个 hook 对象 // // fiber.hooks = [ // { state: 0, queue: [] }, // 第1个 useState // { state: 'Alice', queue: [] }, // 第2个 useState // ] let currentFiber = null let hookIndex = 0 /** * 简化版 useState 实现 * @param {*} initial - 初始值（仅首次渲染使用） * @returns {[*, Function]} [当前 state, setState 函数] */ function useState(initial) { const fiber = currentFiber const idx = hookIndex++ // 首次渲染：初始化 hook 对象 if (!fiber.hooks[idx]) { fiber.hooks[idx] = { state: initial, queue: [] } } const hook = fiber.hooks[idx] // 处理更新队列：将所有待执行的 setState 合并 while (hook.queue.length) { const action = hook.queue.shift() // 支持函数式更新：setState(prev => prev + 1) hook.state = typeof action === 'function' ? action(hook.state) : action } // setState：将新值入队，触发重渲染 const setState = (action) => { hook.queue.push(action) rerender() // 真实 React 中是 scheduleUpdateOnFiber(fiber) } return [hook.state, setState] } // ── 第二步：渲染函数（模拟组件 render）──────────────────────── const fiber = { hooks: [], renderCount: 0 } const log = [] function Counter() { currentFiber = fiber hookIndex = 0 fiber.renderCount++ const [count, setCount] = useState(0) const [text, setText] = useState('hello') log.push(`render #${fiber.renderCount}: count=${count} text="${text}"`) return { count, text, setCount, setText } } function rerender() { Counter() } // ── 第三步：演示 useState 的完整工作流程 ───────────────────── console.log('=== 初始渲染 ===') let comp = Counter() console.log(log.at(-1)) console.log('hooks 数组:', JSON.stringify(fiber.hooks.map(h => ({ state: h.state })))) console.log('\n=== setState(1)：直接赋值 ===') comp.setCount(1) // 入队 → rerender → 从 hooks[0] 读取新值 console.log(log.at(-1)) console.log('\n=== setState(c => c + 1)：函数式更新 ===') comp = Counter() comp.setCount(c => c + 1) console.log(log.at(-1)) console.log('\n=== 多次 setState 合并（批量更新） ===') // 同一轮事件中多次 setState，React 18 会批量处理 // 这里模拟：手动入队多个 action，只触发一次 render fiber.hooks[0].queue.push(10, 20, c => c + 5) // 最终：10 → 20 → 25 Counter() console.log(log.at(-1), '（三次入队，一次渲染，最终 count=25）') console.log('\n=== 为什么 Hooks 不能在条件语句中调用？ ===') // hooks 数组按调用顺序索引，如果某次渲染跳过了一个 useState // 后续所有 hook 的索引都会错位，读到错误的 state console.log('hooks[0]:', fiber.hooks[0].state, '← count') console.log('hooks[1]:', fiber.hooks[1].state, '← text') console.log('顺序固定 → 索引稳定 → state 正确对应')
// TypeScript 类型标注版 interface Hook<T> { state: T queue: Array<T | ((prev: T) => T)> } interface Fiber { hooks: Hook<any>[] renderCount: number } let currentFiber: Fiber | null = null let hookIndex = 0 /** * 简化版 useState 实现 * @param initial - 初始值（仅首次渲染使用） * @returns [当前 state, setState 函数] */ function useState<T>(initial: T): [T, (action: T | ((prev: T) => T)) => void] { const fiber = currentFiber! const idx = hookIndex++ if (!fiber.hooks[idx]) { fiber.hooks[idx] = { state: initial, queue: [] } } const hook = fiber.hooks[idx] as Hook<T> while (hook.queue.length) { const action = hook.queue.shift()! hook.state = typeof action === 'function' ? 5. Hooks 原理实现 面试必考

Hooks 本质：每个组件对应一个 Fiber，Fiber 上有一个 memoizedState 链表，每个 Hook 调用对应链表中的一个节点。 这就是为什么 Hooks 不能在条件语句中调用——顺序必须固定！
let currentFiber = null let hookIndex = 0 /** * 创建模拟 Fiber 节点，用于挂载 Hooks 链表。 * * @returns {{ hooks: Array, renderCount: number }} Fiber 对象 */ function createFiber() { return { hooks: [], renderCount: 0 } } /** * 模拟 React.useState，基于 Fiber hooks 链表实现状态管理。 * * @param {*} initial - 初始状态值或惰性初始化函数 * @returns {[*, Function]} [当前状态, setState 函数] */ function useState(initial) { const fiber = currentFiber const idx = hookIndex++ if (fiber.hooks[idx] === undefined) { fiber.hooks[idx] = { state: typeof initial === 'function' ? initial() : initial, queue: [] } } const hook = fiber.hooks[idx] while (hook.queue.length) { const action = hook.queue.shift() hook.state = typeof action === 'function' ? action(hook.state) : action } const setState = (action) => { hook.queue.push(action); render() } return [hook.state, setState] } /** * 模拟 React.useReducer，通过 reducer 函数管理复杂状态。 * * @param {Function} reducer - 状态归约函数 (state, action) => newState * @param {*} initial - 初始状态 * @returns {[*, Function]} [当前状态, dispatch 函数] */ function useReducer(reducer, initial) { const fiber = currentFiber const idx = hookIndex++ if (fiber.hooks[idx] === undefined) fiber.hooks[idx] = { state: initial, queue: [] } const hook = fiber.hooks[idx] while (hook.queue.length) hook.state = reducer(hook.state, hook.queue.shift()) const dispatch = (action) => { hook.queue.push(action); render() } return [hook.state, dispatch] } /** * 模拟 React.useMemo，依赖未变化时返回缓存值。 * * @param {Function} factory - 计算函数 * @param {Array} deps - 依赖数组 * @returns {*} 缓存或重新计算的值 */ function useMemo(factory, deps) { const fiber = currentFiber const idx = hookIndex++ const prev = fiber.hooks[idx] const hasChanged = !prev || deps.some((d, i) => d !== prev.deps[i]) if (hasChanged) { const value = factory() fiber.hooks[idx] = { value, deps } return value } return prev.value } const fiber = createFiber() const renderLog = [] function render() { currentFiber = fiber; result = render() console.log('setCount(1):', JSON.stringify(renderLog.at(-1))) result.setCount(c => c + 1); result = render() console.log('setCount(c=>c+1):', JSON.stringify(renderLog.at(-1))) result.dispatch({ type: 'INC' }); result.dispatch({ type: 'INC' }); result = render() console.log('dispatch INC×2:', JSON.stringify(renderLog.at(-1))) let memoCalcCount = 0 const fiber2 = createFiber() function renderWithMemo(count) { currentFiber = fiber2; hookIndex = 0 const [c] = useState(count) const expensive = useMemo(() => { memoCalcCount++; return c * 100 }, [c]) return expensive } renderWithMemo(5); renderWithMemo(5); renderWithMemo(5) console.log('useMemo 相同deps，计算次数:', memoCalcCount, '(应为1，后两次缓存)') renderWithMemo(6) console.log('useMemo deps变化，计算次数:', memoCalcCount, '(应为2)') console.log('总渲染次数:', fiber.renderCount)

6. useEffect / useCallback / useRef / useContext 面试必考

副作用与引用 Hooks：useEffect 处理副作用（请求/订阅/DOM 操作），useCallback 缓存函数引用（防止子组件不必要重渲染），useRef 保存跨渲染的可变值（不触发重渲染），useContext 跨层级传递数据。

useContext 原理：React 通过 Context 对象的 Provider 将值注入 Fiber 树，消费组件调用 useContext(ctx) 时，React 沿 Fiber 树向上查找最近的 Provider 取值。Context 变化时，所有消费该 Context 的组件都会重渲染（不受 memo 保护）。

Hooks 使用规则（面试高频）：① 只在函数组件顶层调用（不能在 if/for/嵌套函数中）；② 只在 React 函数组件或自定义 Hook 中调用。原因：Hooks 依赖调用顺序与 Fiber hooks 链表的索引一一对应，顺序改变会导致状态错乱。
