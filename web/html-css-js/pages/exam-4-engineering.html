1 Eng
Vite 为什么比 Webpack 快？核心原理是什么？
参考答案：
开发环境：Vite 利用浏览器原生 ESM 按需编译（请求时才编译），不需要打包整个依赖图；用 esbuild（Go 编写）预构建依赖，比 JS 打包器快 10-100 倍。生产环境：用 Rollup 打包，Tree Shaking 更彻底。HMR：只更新变更模块及其依赖链，不重新构建整个 bundle。
2 Eng
Webpack 中 loader 和 plugin 的区别？
参考答案：
loader 是文件转换器，将非 JS 文件转为 JS 模块（如 babel-loader 转 ES6、css-loader 处理 CSS、file-loader 处理图片）。plugin 是功能扩展，在打包生命周期的各个钩子执行任务（如 HtmlWebpackPlugin 生成 HTML、MiniCssExtractPlugin 提取 CSS、DefinePlugin 定义环境变量）。loader 在 module.rules 配置，plugin 在 plugins 数组配置。
3 Eng
Webpack 的 Tree Shaking 原理？什么情况下会失效？
参考答案：
基于 ES Module 的静态分析，标记未使用的 export 为 unused，然后由 Terser 删除。失效情况：① CommonJS 模块（动态 require）② 有副作用的模块（未配置 sideEffects: false）③ 动态导入 ④ 导出对象的属性（只能整体引入）。解决：使用 ESM、配置 sideEffects、按需导入。

5 Eng
ESLint 和 Prettier 的区别？如何配合使用？
参考答案：
ESLint 检查代码质量和潜在错误（未使用变量、类型错误等）；Prettier 只负责代码格式化（缩进、引号、分号等）。配合：eslint-config-prettier 关闭 ESLint 中与 Prettier 冲突的格式规则，让 Prettier 专管格式，ESLint 专管质量。

husky 管理 Git Hooks + lint-staged 只检查暂存文件 + commitlint 校验提交信息格式。pre-commit 钩子：lint-staged 执行 ESLint + Prettier 格式化。commit-msg 钩子：commitlint 校验 Conventional Commits 格式（feat/fix/docs 等前缀）。

Monorepo 的优缺点？常用工具有哪些？
参考答案：
优点：代码共享、统一版本管理、原子提交、统一 CI/CD。缺点：仓库体积大、构建复杂、权限管理困难。工具：pnpm workspace（推荐）、Turborepo（构建缓存和并行）、Nx（智能构建）、Lerna（版本管理）。

微前端是什么？有哪些实现方案？
参考答案：
将大型前端应用拆分为多个独立开发、部署的子应用。方案：① qiankun（基于 single-spa，JS 沙箱 + CSS 隔离）② Module Federation（Webpack 5 模块联邦）③ iframe（天然隔离但体验差）④ Web Components ⑤ micro-app。选择依据：团队规模、技术栈差异、隔离需求。
9 Perf
Web Vitals 核心指标有哪些？如何优化？
参考答案：
LCP（Largest Contentful Paint）最大内容绘制 < 2.5s：优化关键资源加载、图片优化、SSR。INP（Interaction to Next Paint）交互响应 < 200ms：减少主线程阻塞、代码分割。CLS（Cumulative Layout Shift）布局偏移 < 0.1：预留图片/广告尺寸、避免动态插入内容。

首屏加载优化有哪些手段？
参考答案：
网络层：CDN、HTTP/2、Gzip/Brotli 压缩、预加载（preload/prefetch）、DNS 预解析。资源层：代码分割、Tree Shaking、图片优化（WebP/AVIF、懒加载、响应式图片）。渲染层：SSR/SSG、关键 CSS 内联、骨架屏、字体优化（font-display: swap）。
11 Perf
如何实现虚拟滚动？原理是什么？
参考答案：
只渲染可视区域内的列表项，通过计算滚动位置动态替换 DOM 节点。原理：① 计算可视区域能显示的条目数 ② 监听滚动事件计算起始索引 ③ 只渲染 startIndex ~ endIndex 的数据 ④ 用 padding/transform 撑开容器模拟完整高度。库：react-virtualized、vue-virtual-scroller。

图片优化有哪些手段？

如何减少 JavaScript 执行时间？

15 Perf
HTTP 缓存策略有哪些？如何配置？
参考答案：
强缓存：Cache-Control: max-age=31536000（不发请求）；协商缓存：ETag/If-None-Match 或 Last-Modified/If-Modified-Since（304 响应）。最佳实践：HTML 不缓存或短缓存，JS/CSS/图片带 hash 强缓存一年，API 按业务设置。

XSS 攻击有哪些类型？如何防御？
参考答案：
① 存储型：恶意脚本存入数据库 ② 反射型：URL 参数注入 ③ DOM 型：前端直接操作 DOM 注入。防御：① 输出编码（HTML 实体转义）② CSP（Content-Security-Policy）③ HttpOnly Cookie ④ 避免 innerHTML，用 textContent ⑤ 输入验证和过滤。
17 Safe
CSRF 攻击原理和防御？
参考答案：
利用用户已登录的身份，在第三方网站发起恶意请求。防御：① CSRF Token（服务端生成，前端携带）② SameSite Cookie（Strict/Lax）③ 验证 Referer/Origin ④ 关键操作二次验证。Axios 拦截器统一添加 Token。

前端如何保证接口安全？
参考答案：
① HTTPS 加密传输 ② JWT Token 认证（存 httpOnly Cookie 或内存）③ 请求签名（timestamp + nonce + sign）④ 敏感数据加密（RSA 公钥加密）⑤ 接口限流 ⑥ 参数校验 ⑦ CORS 白名单。

如何实现前端权限控制？
参考答案：
① 路由权限：动态路由（根据角色 addRoute）+ 导航守卫 ② 按钮权限：自定义指令 v-permission 或组件封装 ③ 接口权限：请求拦截器携带 Token，401 跳转登录 ④ 数据权限：后端控制返回数据范围。RBAC 模型：用户→角色→权限。

CI/CD 流程如何设计？
参考答案：
CI（持续集成）：代码提交 → lint 检查 → 单元测试 → 构建 → 产物上传。CD（持续部署）：自动/手动触发 → 部署到测试环境 → 自动化测试 → 部署到生产环境。工具：GitHub Actions、GitLab CI、Jenkins。关键：自动化测试覆盖率、构建缓存、灰度发布。

Service Worker 和 PWA 的作用？
参考答案：
Service Worker 是浏览器后台线程，可拦截网络请求、缓存资源、推送通知。PWA（Progressive Web App）利用 SW 实现：① 离线访问 ② 资源缓存策略（Cache First/Network First/Stale While Revalidate）③ 后台同步 ④ 推送通知 ⑤ 安装到桌面。

如何设计前端埋点方案？
参考答案：
① 代码埋点：手动调用 track() 函数（精确但侵入性强）② 无埋点/全埋点：自动采集点击/曝光事件（覆盖广但数据量大）③ 可视化埋点：通过可视化工具圈选元素配置（灵活但依赖工具）。技术实现：Intersection Observer 监听曝光、事件委托监听点击、sendBeacon 上报、requestIdleCallback 延迟处理。
