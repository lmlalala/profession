试卷二：Vue3 技术栈

Vue3 响应式 · Composition API · Pinia · Vue Router · Vite · TypeScript

1 Vue3
Vue3 的响应式原理是什么？对比 Vue2 的 Object.defineProperty 有哪些优势？
参考答案：
Vue3 使用 Proxy 拦截对象操作。优势：① 支持新增/删除属性检测 ② 支持数组下标和 length ③ 懒代理（访问时才递归）④ 支持 Map/Set/WeakMap ⑤ 性能更好（不需要递归遍历所有属性）。依赖收集：WeakMap→Map→Set 三层结构，track 收集 effect，trigger 触发更新。
2 Vue3
ref 和 reactive 的区别？解构 reactive 对象如何保持响应式？
参考答案：
ref 包装基本类型（通过 .value 访问），也可包装对象；reactive 只能包装对象类型。ref 在模板中自动解包。解构 reactive 会丢失响应式，解决方案：toRefs() 将每个属性转为 ref，或 toRef(obj, 'key') 转单个属性。
3 Vue3
setup 语法糖（script setup）的优势？
参考答案：
① 更简洁，无需 return 暴露变量 ② 更好的 TypeScript 类型推断 ③ 更好的运行时性能（编译为 render 函数的 setup 闭包）④ 支持 defineProps/defineEmits/defineExpose 编译器宏。
4 Vue3
watch 和 watchEffect 的区别？
参考答案：
watch：需显式指定监听源，可获取新旧值，默认懒执行（首次不触发）；watchEffect：自动追踪回调中用到的响应式依赖，立即执行一次，无法获取旧值。watch 适合需要对比新旧值或条件触发的场景，watchEffect 适合副作用同步。
5 Vue3
Vue3 的生命周期钩子有哪些变化？
参考答案：
Options API 的 beforeCreate/created 被 setup 替代；beforeMount→onBeforeMount，mounted→onMounted，beforeUpdate→onBeforeUpdate，updated→onUpdated，beforeUnmount→onBeforeUnmount（Vue2 是 beforeDestroy），unmounted→onUnmounted。新增 onRenderTracked/onRenderTriggered 用于调试。
6 Vue3
defineAsyncComponent 和 Suspense 如何实现组件按需加载？
参考答案：
defineAsyncComponent(() => import('./Comp.vue')) 定义异步组件，配合 Suspense 的 #default 和 #fallback 插槽显示加载状态。Suspense 还支持嵌套异步组件和 async setup()。
7 Vue3
Teleport 和 Fragment 的作用？
参考答案：
Teleport：将组件内容渲染到 DOM 中的其他位置（如 body），适合模态框、通知、tooltip 等需要脱离父组件 DOM 层级的场景。Fragment：组件可以有多个根节点，不再需要包裹 div。
8 Vue3
Vue3 的编译优化有哪些？
参考答案：
① PatchFlag：标记动态节点类型（TEXT/CLASS/STYLE/PROPS），diff 时只比较动态部分 ② 静态提升（hoistStatic）：静态节点只创建一次，复用引用 ③ Block Tree：只收集动态子节点到 dynamicChildren 数组，跳过静态节点遍历 ④ 事件缓存（cacheHandlers）。
9 Vue3
Pinia 与 Vuex 的区别？
参考答案：
① 去掉 mutations，只有 state/getters/actions ② 天然支持 TypeScript ③ 支持多个独立 store（无需 modules 嵌套）④ 支持 Composition API 风格 ⑤ 更轻量（约 1KB）⑥ 支持 SSR ⑦ DevTools 集成。
10 Vue3
Vue Router 4 有哪些新特性？
参考答案：
① Composition API：useRouter()/useRoute() ② 动态路由：addRoute()/removeRoute() ③ 导航守卫返回值替代 next()：return false/return '/path' ④ 支持 TypeScript ⑤ 路由懒加载更简洁。
11 Vue3
组件通信有哪些方式？
参考答案：
① props/emits（父子）② provide/inject（跨层级）③ v-model 双向绑定 ④ $attrs 透传 ⑤ Pinia 全局状态 ⑥ EventBus（mitt 库）⑦ ref/expose 直接访问子组件。
12 Vue3
自定义 Hooks（Composables）的最佳实践？
参考答案：
① 以 use 开头命名 ② 接收 ref 参数保持响应式 ③ 返回 ref 而非 reactive ④ 处理副作用清理（onUnmounted）⑤ 支持 SSR（避免直接访问 window/document）⑥ 单一职责原则。
13 Vue3
v-model 在 Vue3 中有什么变化？
参考答案：
Vue2 的 v-model 等于 :value + @input；Vue3 改为 :modelValue + @update:modelValue。支持多个 v-model：v-model:title="title" v-model:content="content"。自定义修饰符通过 modelModifiers prop 接收。
14 TS
TypeScript 中 interface 和 type 的区别？
参考答案：
interface 可声明合并（同名自动合并）、支持 extends 继承、可被 class implements；type 支持联合类型、交叉类型、条件类型、映射类型，不可声明合并。推荐：对象形状用 interface，其他用 type。
15 TS
泛型的作用？举例说明在 Vue3 项目中的应用。
参考答案：
泛型实现类型参数化。应用：① API 请求封装 function request<T>(url: string): Promise<ApiResponse<T>> ② ref 类型 ref<User | null>(null) ③ 组件 Props 泛型 ④ 通用工具函数如 function pick<T, K extends keyof T>(obj: T, keys: K[]): Pick<T, K>。
16 TS
any、unknown、never 的区别？
参考答案：
any：跳过类型检查，可赋值给任何类型（慎用）；unknown：安全的 any，使用前必须类型收窄（typeof/instanceof/类型守卫）；never：永远不会有值（函数抛异常、死循环、穷尽检查）。
17 TS
如何实现类型收窄？
参考答案：
① typeof 守卫 ② instanceof 守卫 ③ in 操作符 ④ 自定义类型守卫 function isString(x: unknown): x is string ⑤ 可辨识联合（Discriminated Union）通过共同的字面量属性（如 kind/type）区分。
18 TS
常用的工具类型有哪些？
参考答案：
Partial<T>（全可选）、Required<T>（全必选）、Readonly<T>（全只读）、Pick<T,K>（选取）、Omit<T,K>（排除）、Record<K,V>（键值映射）、ReturnType<T>（函数返回类型）、Parameters<T>（函数参数类型）、Extract/Exclude（联合类型提取/排除）。
19 Vue3
如何优化 Vue3 应用性能？
参考答案：
① 合理使用 v-memo/v-once 缓存 ② 大列表用虚拟滚动 ③ 组件懒加载 defineAsyncComponent ④ 合理拆分组件粒度 ⑤ 使用 shallowRef/shallowReactive 减少深层响应式 ⑥ computed 缓存计算结果 ⑦ 避免不必要的 watch。
20 Vue3
Vue3 中如何处理错误？
参考答案：
① 组件级：onErrorCaptured 钩子 ② 全局：app.config.errorHandler ③ 异步错误：try/catch + async setup ④ 路由错误：router.onError() ⑤ Suspense 的 onError 事件。
21 Eng
Vite 为什么比 Webpack 快？
参考答案：
开发环境：Vite 利用浏览器原生 ESM，不需要打包，按需编译（请求时才编译模块）；Webpack 需要先打包整个依赖图。Vite 用 esbuild 预构建依赖（Go 编写，比 JS 快 10-100 倍）。HMR：Vite 只更新变更模块，不需要重新构建整个 bundle。
22 Eng
如何在 Vite 中配置环境变量？
参考答案：
创建 .env / .env.development / .env.production 文件，变量以 VITE_ 前缀命名。代码中通过 import.meta.env.VITE_XXX 访问。vite.config.ts 中通过 loadEnv(mode, process.cwd()) 加载。
23 Vue3
Vue3 的 SSR 原理？Nuxt 3 的优势？
参考答案：
SSR：服务端执行 renderToString() 生成 HTML 字符串，客户端 hydrate 激活事件绑定。Nuxt 3 优势：① 文件系统路由 ② 自动导入 ③ 服务端 API 路由 ④ SSR/SSG/ISR 灵活切换 ⑤ Nitro 服务引擎 ⑥ 丰富的 Modules 生态。
24 Vue3
如何实现 Vue3 的权限控制？
参考答案：
① 路由级：路由 meta 标记权限 + 导航守卫 beforeEach 校验 ② 组件级：自定义指令 v-permission 控制按钮显示 ③ 菜单级：根据角色动态生成菜单（addRoute 动态路由）④ API 级：请求拦截器携带 Token。
25 Vue3
Vue3 中如何实现国际化？
参考答案：
使用 vue-i18n@9，createI18n 配置 locale 和 messages。模板中 $t('key') 或 Composition API 中 useI18n()。支持按需加载语言包（动态 import），支持日期/数字格式化。Nuxt 3 可用 @nuxtjs/i18n 模块。
